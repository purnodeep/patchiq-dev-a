Design Principles

Constraints that shape all four integration patterns

1. Workflows are offered, never required
"No automation" is always the default selected state. Users discover the power without being forced into complexity.
2. Templates show mini inline flow diagrams
Users see the structure before choosing. Small colored dots connected by arrows convey the whole flow in 24px of height.
3. Pill picker for simple automations
Post-deployment actions use a pill picker — simpler than building a full workflow. The workflow diagram is generated from selections.
4. "Open Full Editor" is always one click away
Every inline workflow builder links to the full canvas editor. Power users are never blocked. Casual users never see it unless they want to.
5. Three-color system for node types
Brand (green/purple) for triggers/actions · Amber for gates/approvals · Red for rollback · Gray for end/neutral nodes
6. Plain-English preview summarizes the flow
Below the visual diagram, a natural-language sentence explains what will happen. No jargon. Reviewable before saving.
