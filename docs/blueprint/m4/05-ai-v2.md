# AI v2

**Status**: Planned
**Milestone**: M4
**Dependencies**: M2 MCP server, M3 AI assistant v1, M2 deployment engine, M2 compliance engine

---

## Vision

Move beyond reactive AI assistance to proactive intelligence: predict patch risk before deployment, optimize scheduling around usage patterns, and surface anomalies before they become incidents.

## Deliverables

### Patch Risk Prediction
- [ ] ML model trained on historical deployment data (success rate, rollback rate, OS/app version combos)
- [ ] Risk score per patch-target combination (LOW / MEDIUM / HIGH / CRITICAL)
- [ ] Risk score surfaced in deployment wizard and policy engine
- [ ] Model retraining pipeline: weekly batch job using accumulated deployment outcomes
- [ ] Explainability: top factors contributing to risk score (shown in UI tooltip)

### Optimal Scheduling
- [ ] Agent usage pattern collector: active hours inferred from heartbeat and CPU telemetry
- [ ] Scheduling recommendation: suggest lowest-disruption maintenance window per endpoint group
- [ ] Integration with deployment wizard: "Auto-schedule" option applies recommendation
- [ ] Conflict detection: flag overlapping deployment windows for same endpoint

### Anomaly Detection
- [ ] Baseline model per agent: expected heartbeat cadence, patch success rate, scan duration
- [ ] Alert on deviation: missed heartbeats, unexpected failure spike, scan time outlier
- [ ] Anomaly feed in dashboard (severity-ranked, with last-seen normal state)
- [ ] Suppression rules: exclude known maintenance windows from anomaly scoring

### Natural Language Report Queries
- [ ] NL query interface in Hub and Patch Manager dashboards
- [ ] Example queries: "Show compliance trends for Q4", "Which endpoints failed the most patches last month?"
- [ ] Query translated to API calls via MCP server; results rendered as chart or table
- [ ] Query history saved per user; shareable query links
